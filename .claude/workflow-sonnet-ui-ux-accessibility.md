# Workflow: Sonnet UI/UX & Accessibility Audit

## Agent: Sonnet UI Blueprint & Validator

### Steps
1. Map all user journeys and UI states
2. Specify component usage and registry selection
3. Validate accessibility (WCAG 2.1 AA), keyboard navigation, ARIA
4. Create component hierarchy diagrams
5. Document issues and improvements

### Acceptance Criteria
- Complete user flows and component maps
- Accessibility requirements met
- Keyboard navigation and screen reader support defined
- Issues and recommendations documented
